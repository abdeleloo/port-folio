Correction du bug du bouton bascule multi-domaines

Période : Octobre 2024 – Novembre 2024  |  Contexte : Alternance – IP Twins

Dans l'application métier Domainarium II d'IP Twins, un bug affectait le bouton de bascule permettant de naviguer entre plusieurs domaines d'un portefeuille client. Le comportement était incohérent : le changement de domaine actif ne se reflétait pas correctement dans l'interface, perturbant les opérations des équipes.

1

Identification et analyse du problème

Symptômes observés

  • Le domaine actif ne changeait pas après clic sur le bouton bascule
  • Les données affichées restaient celles du domaine précédent
  • Comportement aléatoire selon l'ordre de navigation
  • Aucune erreur visible dans la console navigateur

Cause identifiée

  • L'identifiant du domaine actif n'était pas mis à jour dans l'état de l'application
  • Le composant affichait l'état mémorisé au chargement de la page
  • L'event listener sur le bouton ne propageait pas correctement la valeur sélectionnée
Vue du portefeuille utilisateur avec le bouton de bascule entre domaines — interface Domainarium II.
2

Correction du code

La correction a porté sur deux points : la mise à jour de l'état actif au clic, et le rechargement des données associées au domaine sélectionné.

Avant – Code problématique

// ❌ AVANT : l'ID du domaine n'est pas transmis au rechargement
document.querySelectorAll('.btn-bascule').forEach(btn => {
  btn.addEventListener('click', () => {
    switchDomain(); // appel sans argument → domaine actif non mis à jour
  });
});

Après – Code corrigé

// ✅ APRÈS : transmission de l'ID du domaine sélectionné
document.querySelectorAll('.btn-bascule').forEach(btn => {
  btn.addEventListener('click', () => {
    const domainId = btn.dataset.domainId;
    switchDomain(domainId); // mise à jour de l'état actif
  });
});

function switchDomain(domainId) {
  // Mise à jour visuelle du bouton actif
  document.querySelectorAll('.btn-bascule').forEach(b => b.classList.remove('active'));
  document.querySelector(`[data-domain-id="${domainId}"]`).classList.add('active');
  // Rechargement des données du domaine sélectionné
  loadDomainData(domainId);
}
Résultat

Après correction, le bouton bascule met bien à jour le domaine actif et recharge les données correspondantes. La correction a été validée et déployée en production.

3

Tests de validation

  • Test de bascule entre plusieurs domaines consécutifs → données correctement mises à jour
  • Test avec un portefeuille à domaine unique → comportement inchangé
  • Test de non-régression sur les autres fonctionnalités de la page portefeuille
  • Validation par l'utilisateur final après déploiement
4
