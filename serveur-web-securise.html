Objectif du Projet

Mise en place d'un serveur Ubuntu sécurisé pour héberger un site web personnel (portfolio) avec Apache, PHP, et un formulaire de contact. Ce projet démontre les compétences en administration système, sécurisation d'infrastructure et déploiement web.

Spécifications Techniques

  • Utilisateur : abde
  • Réseau local : 192.168.1.56
  • Accès SSH : VS Code avec authentification par mot de passe
  • Restriction : Accès SSH uniquement depuis le réseau local
  • Sécurité : Pare-feu UFW configuré et actif
📊

Architecture du Système

Schéma Réseau

Topologie réseau : Client → Routeur → Serveur Ubuntu (192.168.1.56)

Architecture Logicielle

Stack technique : Apache2 → PHP → ModSecurity → UFW/Fail2ban

🔒

Sécurisation du Serveur

Pare-feu UFW

  • Activé avec sudo ufw enable
  • Ports ouverts : 22 (SSH), 80 (HTTP), 443 (HTTPS)
  • Protection brute force SSH : sudo ufw limit 22/tcp

Fail2ban

  • Installé et activé pour sshd
  • Surveillance des tentatives de connexion
  • Vérification : sudo fail2ban-client status sshd

Configuration SSH

  • Accès restreint à l'utilisateur abde
  • Connexion root désactivée
  • Fichier : /etc/ssh/sshd_config
  • Paramètre : PermitRootLogin no

ModSecurity

  • Module : libapache2-mod-security2
  • Configuration : /etc/modsecurity/modsecurity.conf
  • Règles CRS dans /etc/modsecurity/crs/
📚

Configuration Apache

Modules Activés

sudo a2enmod headers
sudo a2enmod rewrite
sudo a2enmod ssl
sudo a2enmod security2

Virtual Host portfolio.local

  • Fichier : /etc/apache2/sites-available/portfolio.conf
  • Chemin du site : /var/www/portfolio
  • Activation :
sudo a2ensite portfolio.conf
sudo systemctl reload apache2

DNS Local

Entrée ajoutée dans C:\Windows\System32\drivers\etc\hosts :

192.168.1.56 portfolio.local
🌐

Déploiement du Site

Installation des Fichiers

Site HTML/CSS/JS copié dans /var/www/portfolio

Configuration des Droits

sudo chown -R abde:www-data /var/www/portfolio
sudo chmod -R 755 /var/www/portfolio

Les permissions permettent à l'utilisateur abde de modifier les fichiers tout en donnant au serveur web les droits de lecture nécessaires.

Monitoring et Surveillance

Commandes de Surveillance

Logs Apache

sudo tail -f /var/log/apache2/access.log
sudo tail -f /var/log/apache2/error.log

Status Fail2ban

sudo fail2ban-client status
sudo fail2ban-client status sshd

Firewall UFW

sudo ufw status verbose
sudo ufw show listening

Processus Apache

sudo systemctl status apache2
sudo apache2ctl status
