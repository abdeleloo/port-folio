Site Web MFC

Période : Septembre 2025 – Décembre 2025  |  Contexte : Formation (école)

Développement d'un site web pour l'entreprise fictive MFC. Ce projet couvre l'ensemble du cycle de développement web : conception, front-end, back-end, base de données et déploiement, dans un contexte de travail collaboratif avec Git.

1

Contexte et analyse du besoin

L'entreprise fictive MFC avait besoin d'un site web vitrine et fonctionnel pour présenter ses services et gérer ses données clients. Le projet a nécessité une phase d'analyse préalable pour définir les fonctionnalités et l'architecture.

  • Rédaction d'un cahier des charges fonctionnel
  • Conception de plan du site
  • Modélisation de la base de données (MCD)
  • Répartition des tâches et planification du projet

Réalisation de l'interface utilisateur responsive, accessible sur mobile et desktop.

Extrait JavaScript – Requête AJAX vers l'API

// Chargement dynamique des données depuis l'API Java
async function loadData(endpoint) {
  try {
    const response = await fetch(`/api/${endpoint}`);
    if (!response.ok) throw new Error(`Erreur HTTP: ${response.status}`);
    const data = await response.json();
    renderTable(data);
  } catch (error) {
    showAlert('Impossible de charger les données.', 'error');
    console.error(error);
  }
}

// Affichage dynamique dans le tableau HTML
function renderTable(data) {
  const tbody = document.querySelector('#dataTable tbody');
  tbody.innerHTML = data.map(item => `
    <tr>
      <td>${item.id}</td>
      <td>${item.nom}</td>
      <td>${item.valeur}</td>
    </tr>
  `).join('');
}

Base de données SQL

Conception et implémentation de la base de données relationnelle.

-- Structure principale de la base MFC
CREATE TABLE clients (
    id       INT PRIMARY KEY AUTO_INCREMENT,
    nom      VARCHAR(100) NOT NULL,
    email    VARCHAR(150) UNIQUE NOT NULL,
    tel      VARCHAR(20),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE commandes (
    id         INT PRIMARY KEY AUTO_INCREMENT,
    client_id  INT NOT NULL,
    montant    DECIMAL(10,2) NOT NULL,
    statut     ENUM('en_cours','validee','annulee') DEFAULT 'en_cours',
    date_cmd   DATE NOT NULL,
    FOREIGN KEY (client_id) REFERENCES clients(id) ON DELETE CASCADE
);
