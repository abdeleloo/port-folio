Rectification du comportement d'affichage des détails de facturation

Période : Janvier 2026 – Février 2026  |  Contexte : Alternance – IP Twins

Un bug affectait le bouton dédié à l'affichage des détails de facturation dans l'application interne d'IP Twins. Le comportement était incohérent : les données ne s'affichaient pas correctement selon les cas, perturbant le travail des équipes de gestion.

1

Identification et analyse du problème

Le problème a été remonté par un utilisateur interne : en cliquant sur le bouton "Voir les détails" d'une facture, le panneau de détails s'affichait parfois vide, parfois avec les données d'une facture précédente.

Symptômes observés

  • Panneau de détails vide au premier clic
  • Données d'une précédente facture persistantes
  • Comportement aléatoire selon la rapidité du clic
  • Aucune erreur visible dans la console navigateur

Cause identifiée

  • Requête AJAX asynchrone non attendue (pas de gestion de la promesse)
  • Le DOM était mis à jour avant la réponse du serveur
  • Absence de réinitialisation des données entre deux clics
  • Event listener dupliqué sur certains boutons
Note

Les fichiers de sortie affectés par le bug étaient corrompus et ne sont plus accessibles. La documentation du problème repose sur l'analyse du code et les symptômes décrits ci-dessus.

2

Correction du code

Trois corrections ont été apportées pour résoudre le comportement erratique :

Avant – Code problématique

// ❌ AVANT : mise à jour du DOM avant la réponse
function showDetails(factureId) {
  updatePanel({}); // réinitialisation immédiate mais DOM déjà peuplé
  fetch(`/api/factures/${factureId}`)
    .then(r => r.json())
    .then(data => updatePanel(data));
  // problème : event listener ajouté plusieurs fois
  btn.addEventListener('click', () => showDetails(id));
}

Après – Code corrigé

// ✅ APRÈS : async/await + loader + event listener unique
async function showDetails(factureId) {
  // Réinitialiser le panneau et afficher un loader
  resetPanel();
  showLoader(true);

  try {
    const response = await fetch(`/api/factures/${factureId}`);
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    const data = await response.json();
    updatePanel(data);
  } catch (err) {
    showError('Impossible de charger les détails.');
    console.error(err);
  } finally {
    showLoader(false);
  }
}

// Event listener unique avec délégation d'événement
document.querySelector('#facturesContainer').addEventListener('click', e => {
  const btn = e.target.closest('[data-facture-id]');
  if (btn) showDetails(btn.dataset.factureId);
});
Résultat de la correction

Après correction, le panneau de détails s'affiche correctement à chaque clic, sans données résiduelles ni comportement aléatoire. La correction a été validée par l'utilisateur final avant déploiement en production.

3

Tests de validation

  • Test de clics rapides successifs → plus de données résiduelles
  • Test avec connexion lente simulée (DevTools > Slow 3G) → loader affiché correctement
  • Test de non-régression sur les autres fonctionnalités de la page
  • Validation par l'utilisateur final après correction
4

Bilan et compétences acquises

Points positifs

  • Maîtrise du débogage JavaScript asynchrone
  • Bonne compréhension de l'event delegation
  • Amélioration de l'UX (loader, gestion d'erreur)
  • Correction validée et déployée en production

Axes d'amélioration
